# 小游戏开发标准 Prompt

## 项目结构

```
website/
├── index.html          # 导航中心首页
├── style.css           # 全局样式
├── script.js           # 全局脚本
└── [game-name]/        # 游戏目录
    └── index.html      # 游戏页面（单文件包含所有代码）
```

4. **反馈机制**: [得分/提示/错误提示/胜利条件]

### 技术规范
- **单文件开发**: 所有 HTML/CSS/JavaScript 写在一个 `index.html` 文件中
- **响应式设计**: 适配桌面端（1200px+）和移动端（<600px）
- **浏览器兼容**: 支持现代浏览器（Chrome/Firefox/Safari/Edge）
- **无外部依赖**: 不依赖任何第三方库，纯原生实现

### 页面布局要求

#### 三栏布局（桌面端）
```
┌─────────────┬──────────────┬─────────────┐
│  左侧边栏    │   游戏主体    │  右侧边栏    │
│  280px      │   自适应      │  280px      │
│             │              │             │
│ • 规则说明   │ • 游戏标题    │ • 益处介绍   │
│ • 起源背景   │ • 控制按钮    │ • 趣味故事   │
│ • 趣闻轶事   │ • 游戏区域    │ • 相关知识   │
│             │ • 数据统计    │             │
└─────────────┴──────────────┴─────────────┘
```

#### 移动端适配
- 侧边栏自动隐藏（`display: none`）
- 游戏主体占满屏幕
- 触发条件: `@media (max-width: 1200px)`

### 内容要求

### 样式规范

#### 配色方案
```css
/* 主色调 */
--primary-color: #667eea;      /* 主题紫蓝色 */
--primary-hover: #5568d3;      /* 悬停色 */
--secondary-color: #764ba2;    /* 渐变辅助色 */

/* 背景 */
--bg-gradient: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
--card-bg: rgba(255, 255, 255, 0.95);

/* 文字 */
--text-primary: #333;
--text-secondary: #666;
--text-light: #999;

/* 状态色 */
--success: #d4edda;
--error: #ffcccc;
--warning: #fff3cd;
```

#### 组件样式
```css
/* 按钮 */
.btn-primary {
    background: #667eea;
    color: white;
    padding: 12px 24px;
    border-radius: 8px;
    transition: all 0.3s;
}

/* 卡片 */
.sidebar {
    background: rgba(255, 255, 255, 0.95);
    border-radius: 15px;
    padding: 20px;
    box-shadow: 0 10px 30px rgba(0,0,0,0.2);
}

/* 故事盒子 */
.story-box {
    background: #f8f9fa;
    padding: 12px;
    border-radius: 8px;
    border-left: 3px solid #667eea;
}
```

### 交互要求
1. **按钮反馈**: 悬停时上移 2px + 颜色变化
2. **选中状态**: 明显的视觉反馈（背景色/边框）
3. **错误提示**: 红色高亮 + 文字提示
4. **成功提示**: 绿色背景 + 庆祝文案
5. **加载状态**: 必要时显示加载动画

### 用户体验
1. **即时反馈**: 每次操作都有视觉/听觉反馈
2. **容错设计**: 允许撤销/重置操作
3. **提示系统**: 提供帮助按钮和提示功能
4. **进度显示**: 显示当前进度和统计数据
5. **无障碍**: 支持键盘操作

### 性能要求
- 首屏加载 < 1s
- 交互响应 < 100ms
- 动画流畅 60fps
- 内存占用 < 50MB

## 开发流程

### 1. 创建游戏目录
```bash
mkdir website/[game-name]
```

### 2. 开发游戏页面
创建 `website/[game-name]/index.html`，包含：
- 完整的 HTML 结构
- 内联 CSS 样式
- 内联 JavaScript 逻辑

### 3. 更新导航页
在 `website/index.html` 中添加游戏卡片：
```html
<a href="[game-name]/index.html" class="project-card">
    <div class="card-icon">[emoji]</div>
    <h3>[游戏名称]</h3>
    <p>[游戏描述]</p>
    <span class="card-tag">[标签]</span>
</a>
```

### 4. 测试验证
- [ ] 桌面端显示正常（Chrome/Firefox/Safari）
- [ ] 移动端适配正确（侧边栏隐藏）
- [ ] 游戏逻辑无 bug
- [ ] 交互流畅无卡顿
- [ ] 内容准确无错别字

### 5. 提交发布
```bash
git add .
git commit -m "Add: [游戏名称] - [简短描述]"
git push origin main
```

## 常见游戏类型参考

### 益智类
- 数独、扫雷、2048、华容道
- 特点：逻辑推理、策略规划

### 反应类
- 打地鼠、接水果、躲避障碍
- 特点：手眼协调、快速反应

### 记忆类
- 翻牌配对、记忆序列、找不同
- 特点：记忆力训练、专注力

### 教育类
- 数学运算、拼音练习、英语单词
- 特点：寓教于乐、知识学习

### 创意类
- 画板涂鸦、音乐创作、故事接龙
- 特点：创造力、想象力

## 注意事项

1. **版权合规**: 确保所有内容（文字/图片/音效）无版权问题
2. **适龄设计**: 根据目标用户调整难度和内容
3. **无障碍**: 考虑色盲用户，不仅依赖颜色区分
4. **隐私保护**: 不收集用户数据，不使用 Cookie
5. **离线可用**: 尽量支持离线游玩（Service Worker 可选）
